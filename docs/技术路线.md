# ClawGuard Web 技术路线

系统定位为 OpenClaw 外层安全代理，不修改 OpenClaw 内核。所有任务必须先进入 Web API，再依次经过身份认证、CFL 签名校验、输入过滤、命令过滤、PBAC 权限判断、Docker 沙箱执行和审计记录。

## 分层

1. 前端交互层：登录、任务提交、任务结果、审计、策略、告警。
2. Web API 层：统一入口和调度。
3. 安全控制层：JWT、CFL、Nonce、防重放、过滤器、PBAC。
4. 沙箱执行层：Docker 运行 OpenClaw stub 或真实 OpenClaw。
5. 审计监控层：脱敏日志、风险事件和告警。

## 防护链

用户 -> API -> JWT/CFL -> 输入过滤 -> 命令过滤 -> PBAC -> Docker -> 审计 -> 前端。
